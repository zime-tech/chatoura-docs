Upload documents to train your AI agent on product manuals, FAQs, policies, and other business content.

## Supported Formats

| Format | Extension | Notes |
|--------|-----------|-------|
| PDF | .pdf | Text-based PDFs only |
| Word | .docx | Microsoft Word documents |
| Text | .txt | Plain text files |

<Note>
Scanned PDFs (images) are not currently supported. Ensure PDFs contain selectable text.
</Note>

## Uploading Documents

1. Go to **Knowledge Base** > **Documents**
2. Click **Upload** or drag files into the upload area
3. Wait for processing to complete
4. Click **Retrain** to update your agent

## Processing Status

| Status | Meaning |
|--------|---------|
| Processing | File is being analyzed |
| Active | Successfully processed and included in training |
| Error | Processing failed - check error message |

## File Size Limits

- Maximum file size: 10 MB per file
- Maximum files: Depends on plan
- Total characters: Subject to plan limits

## Viewing Content

Click on any document to:
- View extracted text
- Check character count
- See processing date
- Delete if needed

## Tips for Better Results

### Document Preparation
- Use clear headings and structure
- Remove irrelevant content before uploading
- Ensure text is properly formatted
- Split very large documents into logical sections

### Content Quality
- Include complete information
- Use natural language
- Add context where needed
- Avoid abbreviations without explanation

## Troubleshooting

<AccordionGroup>
  <Accordion title="Document shows 'Error' status">
    - Check if the PDF contains selectable text
    - Ensure the file isn't corrupted
    - Try re-uploading the document
    - Contact support if the issue persists
  </Accordion>
  <Accordion title="Agent doesn't know document content">
    - Verify the document status is "Active"
    - Click **Retrain** after uploading
    - Test with questions directly from the document
  </Accordion>
  <Accordion title="Character count seems wrong">
    - Only text content is counted
    - Images, headers/footers may be excluded
    - Complex formatting may affect extraction
  </Accordion>
</AccordionGroup>
